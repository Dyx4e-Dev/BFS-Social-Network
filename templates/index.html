BFS Complexity Analysis

⏱ BFS Algorithm

Time Complexity O(V + E)
Space Complexity O(V) 
V = number of nodes (vertices)
E = number of edges

Time: Each node is enqueued exactly once → O(V).
Each edge is examined twice → O(E).
Total: O(V + E).

Space: Queue holds at most V nodes.
Visited set holds at most V entries.
Total extra space: O(V).

🗂 Graph (Adjacency List)

Storage O(V + E)
Add node O(1)
Add edge O(1)
Remove node O(V + E)
Get neighbors O(1)

📭 Queue (collections.deque)

Enqueue O(1)
Dequeue O(1)
Peek (front) O(1)
Space O(V)
Python's collections.deque is a doubly-linked list providing O(1) appends and popleft — ideal for BFS.

🔍 Why BFS?

BFS explores the graph level by level — exactly how information spreads through a social network.

📄 BFS Pseudocode

BFS(Graph G, source s):
  queue   ← deque([s])
  visited ← {s}
  level[s] ← 0

  while queue is not empty:
    u ← queue.popleft()    # dequeue

    for each neighbor v of u:
      if v not in visited:
        visited.add(v)
        level[v] ← level[u] + 1
        queue.append(v)   # enqueue

  return visited, level
